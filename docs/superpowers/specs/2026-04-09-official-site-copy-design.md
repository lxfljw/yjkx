# 有迹可循官网文案与信息架构设计（基于 official-site 现状实现）

## 1. 目标与范围
- 目标用户：家庭主收纳人（家庭内主要负责收纳与找物的人）。
- 转化策略：先建立认知，再引导体验。
- 首屏风格：情绪价值型主标题。
- 本文档范围：仅定义官网单页的信息架构与文案策略，基于当前 `official-site` 已实现代码进行对齐与微调建议。

## 2. 实现现状对照（代码基线）

### 2.1 页面结构
当前首页已按单页串联实现，入口文件：
- `official-site/app/page.tsx`

已接入区块与顺序：
1. `Header`
2. `Hero`
3. `Pain`
4. `Features`
5. `Demo`
6. `Flow`
7. `Value`
8. `Faq`
9. `Cta`
10. `Footer`

组件目录：
- `official-site/app/components/*.tsx`

### 2.2 已实现的核心行为
- 顶栏锚点导航：`#features`、`#flow`、`#faq`（`Header.tsx`）。
- 多处体验外链统一到同一 H5 地址（各组件内 `EXPERIENCE_URL` 一致）。
- 演示区支持横向滚动 + 点击放大遮罩（`Demo.tsx`，`useState` 控制 lightbox）。
- FAQ 使用 `<details>/<summary>` 折叠（`Faq.tsx`）。
- 页面为移动优先的响应式布局（Tailwind 断点 `md:`）。

### 2.3 与当前目标存在的偏差
- 信息顺序偏差：当前是 `Features -> Demo -> Flow -> Value`，而本次目标建议 `Value -> Features -> Flow -> Demo`。
- 首屏按钮主次偏差：当前 Hero 是“立即体验”在前、“了解能力”在后；本次策略希望“了解能力”为主操作。
- 全局元信息偏差：`layout.tsx` 仍是默认 `Create Next App` 元信息，未与业务标题对齐。

## 3. 信息架构方案（确认版）
采用“情绪递进型（A）”，单页顺序定义如下：
1. Hero（情绪共鸣）
2. Pain（问题共鸣）
3. Value（先承诺变化）
4. Features（再解释能力）
5. Flow（上手路径）
6. Demo（视觉证明）
7. FAQ（疑虑消解）
8. CTA（集中转化）

设计理由：
- 对家庭主收纳人，先建立“被理解”感受，再给能力解释，转化链路更自然。
- “先认知后转化”场景下，减少用户在前两屏的决策压力。

## 4. 文案框架（区块级）

### 4.1 Hero
- 主标题：告别翻箱倒柜，把家还给生活
- 副文案：用“空间拍照 + 物品标记”，把每件东西放在哪都记录清楚；需要时一搜就知道。
- 按钮主次：主按钮 `了解能力`，次按钮 `立即体验`

### 4.2 Pain
- 标题：这些找东西时刻，你一定不陌生
- 三个痛点：
  - 明明在家却找不到
  - 找不到就重复买
  - 家人总问“放哪了”

### 4.3 Value
- 标题：先把“混乱感”降下来
- 三个价值：
  - 找物时间更短
  - 重复购买更少
  - 家人沟通成本更低

### 4.4 Features
- 标题：三个能力，撑起家庭收纳记忆
- 三个能力：
  - 空间拍照建档
  - 物品位置标记
  - 拿走放回轨迹可查

### 4.5 Flow
- 标题：上手只要四步
- 四步路径：建空间 -> 标物品 -> 搜索定位 -> 移动后更新

### 4.6 Demo
- 标题：看一眼就知道怎么用
- 说明：展示 5 张截图，支持横向浏览与点击放大查看。

### 4.7 FAQ
- 标题：你可能还关心
- 问题方向：多人使用、准确性、安全性、收费规则。

### 4.8 CTA
- 标题：先把一个抽屉试起来
- 副文案：从最常用区域开始，今天就建立第一份家庭物品地图。
- 按钮：立即体验

## 6. 验收标准
- 单页顺序符合第 3 节定义。
- Hero 主按钮为“了解能力”，次按钮为“立即体验”。
- 至少 3 处“立即体验”入口保持同一链接。
- Demo 区保持“横向滚动 + 点击放大”。
- FAQ 保持折叠展开交互。
- 页面文案语气统一为“家庭主收纳人 + 情绪价值先行”。

## 7. 风险与约束
- `layout.tsx` 与 `page.tsx` 均定义 metadata 时，需统一来源避免 SEO 混乱。
- 当前全局 `globals.css` 保留深色模式变量，但页面主要使用浅色设计；若未来启用深色主题需统一规范。
- 体验链接分散在多个组件常量中；后续可收敛到共享常量，但不在本轮范围。

## 8. Spec 自检
- 无 TBD/TODO 占位。
- 信息架构、文案框架、验收标准一致，无互相冲突。
- 范围聚焦在单页文案与结构，不扩展到新功能。
- 模糊项已收敛为明确文件边界与验收条件。
